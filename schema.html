The required dropbox format is a *.eln format which is basically a zipped RO-Crate with a .eln file extension. For the ro-crate-metadata.json file, follows the RO-Crate 1.1+ Specification, the following structure is expected:

PropertiesÔÉÅ

  • Dataset

    • identifier: Text or URL (required) on the root element level the PERMID of the object in openBIS, which is linked to the OMERO data

    • additionalType: Text (required) openBIS object type

    • description: Text (optional)

    • author: Person (optional)

    • creator: Person (required) user in OMERO and openBIS that links the sources together

    • dateCreated: Date or DateTime (required) creation data of the linkage

    • dateModified: Date or DateTime (required)

    • name: Text (required) on the linked dataset level the name for the ENTRY object in openBIS that will be created

    • text: Text (optional)

    • hasPart: Dataset (optional) data from OMERO

  • Person

    • givenName: Text (required)

    • familyName: Text (required)

    • alternateName: Text (required) userID in openBIS

  • SoftwareApplication

    • installUrl: URL (optional)

    • name: Text (required)

    • softwareVersion: Text (optional)

  • CreateAction , see also Provenance ro-crate

    • object(required)

    • name(optional)

    • instrument(required)

    • actionStatus(optional)

    • endTime(optional)
